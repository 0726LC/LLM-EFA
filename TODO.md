## M1. 数据与预处理脚手架（第 1-2 周）

### 你要做什么
- [ ] 建立 `data/README.md`，定义数据来源、字段、时间粒度、时区、缺失值策略
- [ ] 放入 **可公开获取的样例数据**（例如天气、负荷、节假日特征）
- [ ] 提供数据下载脚本（不直接上传受限原始数据）
- [ ] 提供最小可运行预处理脚本（输出训练所需标准化表格）

### 推荐目录结构
```text
LLM-EFA/
  data/
    README.md
    sample/
  scripts/
    download_data.py
    preprocess.py
  configs/
    data.yaml
```

### 验收标准
- [ ] 新用户按文档可以下载/整理出同结构数据
- [ ] `preprocess` 脚本可生成统一格式文件（如 parquet/csv）

## M2. 论文关键结果复现（第 2-3 周）

### 你要做什么
- [ ] 公布论文中的关键实验设置（数据切分、评价指标、对比方法）
- [ ] 发布“轻量复现”版本：先复现 1-2 个核心表格/图

### 建议先放的内容

### 验收标准
- [ ] 读者可在有限资源下复现至少一个核心结果
- [ ] 与论文报告数值误差在可接受范围内（例如 ±1~3%）

## M4. 一键复现与质量完善（第 5-6 周）

### 你要做什么
- [ ] 增加 `requirements.txt` 或 `environment.yml`
- [ ] 增加 `Makefile` / `run.sh`（一键执行数据→训练→评估）
- [ ] 增加单元测试（数据处理、指标计算、推理接口）
- [ ] 增加 CI（至少跑 lint + 单测）
- [ ] 增加 FAQ：常见报错、复现偏差解释

### 验收标准
- [ ] 新用户可按文档在 1-2 小时内完成最小复现
- [ ] 每次提交自动检查基本质量
